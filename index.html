Core Capabilities

Everything you need to turn Discord into a secure, searchable object store.

Chunked Uploads

Archives are split into 9.5MB chunks to respect Discord limits, with resumable uploads for large data.

Strong Encryption

PBKDF2-derived keys with per-batch salts and AES-256 (Fernet) encryption keep data private.

Compression First

Files are bundled into a .tar.gz archive before encryption to maximize storage efficiency.

Resumable Transfers

Resume stalled uploads with resume and verify integrity with SHA-256.

Metadata & Search

SQLite with WAL and indexes for fast batch search, detailed stats, and sync from Discord.

Friendly CLI

Simple CLI for uploads, downloads, listing, info, and stats—built for non-technical users.

Web UI Dashboard

FastAPI-powered Web UI with live job status, uploads, batch browsing, and system actions.

The Workflow

Package & Encrypt

The CLI bundles your files into a .tar.gz, encrypts with PBKDF2-derived keys, then splits into upload chunks.

1
> Scanning folder...
> Creating archive.tar.gz...
> Encrypting and splitting...
$ python bot.py upload ./data
[Bot] Creating batch card...
[Bot] Uploading 190 chunks...
2

Upload to Discord

Use python bot.py upload. The bot posts a batch card, creates a thread, and uploads chunks concurrently.

Download & Restore

Use python bot.py download <batch_id>. Chunks are verified, merged, decrypted, and extracted.

3
> Verifying hashes... OK
> Decrypting archive... OK
> Extracted: ./data

Installation & Configuration

1. Create Virtual Env & Install

git clone https://github.com/Mo-Shakib/discord-object-store.git
cd discord-object-store
python -m venv .venv
source .venv/bin/activate  # or .venv\Scripts\activate on Windows
pip install -r requirements.txt

2. Run Setup Wizard

Let the wizard save your token, generate keys, and configure channels.

python setup.py

CLI Commands

All commands run in the CLI. This version has no Discord chat commands.
Command Description
python bot.py upload <path> Upload a file or folder, create a batch card, and store chunks in a thread.
python bot.py download <batch_id> Download and reconstruct a batch to a local path.
python bot.py resume <batch_id> Resume an interrupted upload.
python bot.py stats Show storage statistics and recent batch info.
python bot.py list List batches with metadata from the local database.
python bot.py verify <batch_id> Verify integrity with SHA-256 and report missing chunks.

v3 Highlights

  • SQLite Metadata: WAL mode, indexes, and fast local search.

  • Sync & Backup: Rebuild DB from Discord and optional DB backups.

Note: Keep your encryption key secure. Without it, batches cannot be decrypted.

v2.0.0

Major security overhaul and architecture improvements.

New Security Model

  • Encryption: AES-256-GCM per chunk.
  • Key Derivation: PBKDF2-HMAC-SHA256 with 600,000 iterations.
  • Salt/Nonce: Per-chunk random 16-byte salt + 12-byte nonce.
  • Compression: Gzip level 9 applied before encryption.

Important: If USER_KEY is not set, the slicer refuses to create chunks. The bot will upload unencrypted files only if manually placed in the upload folder.

Initial release with basic chunking, upload/download commands, and archive cards.

Security Notice

Security is handled client-side via the python scripts. Your Discord Token is used for transport, but your USER_KEY is used for encryption. Without the key, the files stored on Discord are just random binary blobs. Always back up your key and your local log file.
