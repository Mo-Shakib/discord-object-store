Core Capabilities

Everything you need to turn a chat channel into a robust file system.

Smart Chunking

Files are automatically split into ~9.5MB chunks to respect Discord's attachment limits, ensuring strict compliance and reliability.

AES-256 Encryption

Your data is yours. PBKDF2 key derivation and AES-256-GCM encryption ensure files are unreadable without your passphrase.

High Compression

Built-in Gzip (Level 9) compression before encryption maximizes storage efficiency and speeds up transfers.

Resumable Transfers

Internet dropout? No problem. The !resume command picks up exactly where it left off, skipping already verified chunks.

Metadata & Search

Archive cards provide rich metadata. Search your "drive" instantly with !archives [query] directly from Discord.

Pure Python

Built on discord.py and standard cryptography libraries. Easy to audit, easy to modify, and cross-platform.

The Workflow

Slice & Encrypt

Run slice_and_assemble.py. It processes your folder, compresses files, encrypts them, and splits them into .bin chunks plus a JSON manifest.

1
> Processing video.mp4 (2GB)...
> Chunk 1: encrypted (Salt: x82a...)
> Chunk 2: encrypted
[Bot] !upload
[Bot] Creating Thread: Archive #001
[Bot] Uploading 205 files...
2

Upload to Discord

Use !upload. The bot creates an Archive Card and a dedicated thread, uploading chunks concurrently. Local chunks are deleted upon success to save space.

Download & Restore

Use !download #Archive-ID. The bot fetches the attachments. Run the assembler script to decrypt, decompress, and reconstruct your original files perfectly.

3
> Verifying Manifest... OK
> Decrypting chunk 1... OK
> Reassembled: video.mp4

Installation & Configuration

1. Clone and Install Dependencies

git clone https://github.com/Mo-Shakib/discord-object-store.git
cd discord-object-store
python -m venv .venv
source .venv/bin/activate  # or .venv\Scripts\activate on Windows
pip install -r requirements.txt

2. Configure .env

Create a .env file in the root directory.

DISCORD_BOT_TOKEN=your_bot_token_here
DISCORD_DRIVE_ARCHIVE_CHANNEL_ID=123456789012345678
USER_KEY=your-strong-passphrase-for-encryption

# Optional: Paths (Defaults to OS specifics)
DISCORD_DRIVE_UPLOAD_PATH=./uploads
DISCORD_DRIVE_DOWNLOAD_PATH=./downloads

Bot Commands

Command Description
!upload Upload all .bin and manifest files currently in the upload folder to a new archive thread.
!download #ID Download an archive by its ID (e.g., #120225-01). Supports ranges like [#A..#B].
!resume [ID] Retry missing chunks for the last failed archive or a specific ID.
!status Show drive stats, queue size, encryption status, and last archive info.
!archives [q] List or search recent archives by ID or filename query.
!verify #ID Verify that all chunks exist in the archive thread.

Change Log

v2.0.0 Latest

Major security overhaul and architecture improvements.

New Security Model

  • Encryption: AES-256-GCM per chunk.
  • Key Derivation: PBKDF2-HMAC-SHA256 with 600,000 iterations.
  • Salt/Nonce: Per-chunk random 16-byte salt + 12-byte nonce.
  • Compression: Gzip level 9 applied before encryption.

Important: If USER_KEY is not set, the slicer refuses to create chunks. The bot will upload unencrypted files only if manually placed in the upload folder.

v1.0.0

Initial release with basic chunking, upload/download commands, and archive cards.

Security Notice

Security is handled client-side via the python scripts. Your Discord Token is used for transport, but your USER_KEY is used for encryption. Without the key, the files stored on Discord are just random binary blobs. Always back up your key and your local log file.
